# clean the data and convert "entry" from char to logical.
NYC_Transit_tidy <- NYC_Transit %>% 
  janitor::clean_names() %>% 
  mutate(entry = recode(entry, 'YES' = TRUE, 'NO' = FALSE)) %>% 
  mutate(route8 = as.character(route8), route9 = as.character(route9),
         route10 = as.character(route10), route11 = as.character(route11)) %>% 
  pivot_longer(route1:route11, names_to = "route_number",
               names_prefix = "route", values_to = "route_name") %>% 
  drop_na(route_name)

Answer:

This dataset contains 23 variables, including 16 character variables,
e.g. division, line,4 numeric variables, e.g. station latitude, station
longitude, and 3 logical variables, e.g. ada, free crossover, entry.

After I read in the data, I used clean_names() to make the all the
variables name lowercase, and then I mutated the entry variable from
character to logical type. I converted route8-route11 to character
variable, since route variable spread over the table, so I made the
route1 to route11 to be a new route_number variable while the value
under each route made a new value variable route_name. Then I deleted
the prefix of route1 to route11 in order to only contain numbers. In the
end, I removed all the rows which contains NA in route_name (where the
subway does not have that routename, e.g. N/R).

There are 4270 rows and 23 columns. The data can be considered tidy,
since each variable has its own column and each observation has its own
row while each value in its cell.
